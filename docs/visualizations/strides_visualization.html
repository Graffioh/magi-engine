nested structure — strides come from the geometry

Each outer box holds one full sub-tensor. The size of a sub-tensor (in elements) is the stride of the axis that selects it. That's why the formula is strides[i] = strides[i+1] × shape[i+1] — each step out adds another layer of "how many cells are inside me."

flat memory — see the jumps

Arcs show "+1 along axis k" jumps starting from the selected cell. The length of each arc is the stride of that axis.

the code

// shape.size() == n. strides also has size n.
strides.resize(shape.size(), 1);                // every stride starts at 1
for (int i = (int)strides.size() - 2; i >= 0; --i) {
    strides[i] = strides[i + 1] * shape[i + 1];
}

⚠️ A size_t loop with i >= 0 is an infinite loop — unsigned types never go below zero. Use signed int or the i-->0 trick.
